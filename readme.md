Сервис онлайн ресторанного меню  (без веб морды)
Что должен уметь
1) Регистрация/редактирование заведения. Предполагаемый минимум по информации это название, ссылка на CDN с логотипом
2) Создание меню/редактирование существующего. Предполагаем что 1 заведение 1 меню, для каждлого блюда уникальный id оп которому его можно редакитровать либо удалить. Минимальный базовый набор данных для него это название, состав, цена за единицу 
3) Формирование корзины. Получает id блюд  и по запросу возвращает текущий состав корзины, предполагаем что в корзину можно только добавлять блюда. Считаем что id корзины = id заказа
4) Завершение заказа. Закрытие текущей корзины и возврат итоговой суммы заказа
5) Получение инофрмации по заказу. По id заказа


Из важного 
Информация по заведениям, меню и заказам должна сохранятся локально. Связка между сущностями обя
Доступ к инфе о конкретном заведении должен быть изолирован от остальных (авторизация)



Структура данных в грубой форме по сущностям
Ресторан
  Id
  Название 
  Лого

Блюдо 
  id
  id ресторана
  Название
  Состав
  Цена
  Тип (опциональное)

Корзина 
  id
  id ресторана
  Список id блюд
  Статус (Закрыт/Открыт)
  Итоговая сумма (проставляется при закрытии)
